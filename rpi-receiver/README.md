# IIS3DWB UDP 수신기 (라즈베리파이)

ESP32-S3가 WiFi(UDP)로 보내는 진동 센서 데이터를 라즈베리파이에서 수신·저장합니다.

> 동작 구조: **센서(ESP32-S3)** 는 USB로 전력만 받고, 부팅 시 자동으로
> NVS에 저장된 `server_ip:server_port` 로 진동 데이터를 UDP 전송합니다.
> **라즈베리파이** 는 같은 네트워크에서 그 포트를 열고 수신·기록합니다.

---

## 1. 라즈베리파이에서 설치 (git clone)

```bash
git clone https://github.com/JamesjinKim/3dbw.git
cd 3dbw/rpi-receiver
chmod +x run.sh        # 최초 1회
```

추가 설치 불필요 — Python 3.6+ 표준 라이브러리만 사용합니다.

## 2. 수신기 실행

```bash
./run.sh                 # 포트 9000, raw 값으로 vibration.csv 저장
./run.sh --mg --fs 4     # mg 단위로 변환 저장 (풀스케일 ±4g)
PORT=9001 ./run.sh       # 포트 변경
```

실행하면 **이 라즈베리파이의 IP**가 표시됩니다:

```
====================================================
  IIS3DWB UDP 수신기
====================================================
  이 라즈베리파이 IP : 192.168.0.37
  수신 포트          : 9000

  ▶ 디바이스(ESP32) NVS 에 아래 값을 넣으세요:
      server_ip   = 192.168.0.37
      server_port = 9000
====================================================
[수신 대기] … (Ctrl+C 종료)
```

## 3. 디바이스(ESP32)에 이 IP를 주입

위에서 표시된 `server_ip` / `server_port` 를 디바이스 NVS에 저장합니다.
(설정 툴 또는 NVS 직접 주입 — 펌웨어 `config_manager` 의 `srv_ip`/`srv_port`/`stream_rate`)

| NVS 키 | 값 | 설명 |
|--------|-----|------|
| `srv_ip` | 192.168.0.37 | 라즈베리파이 IP (위에서 확인) |
| `srv_port` | 9000 | 수신 포트 |
| `stream_rate` | 0 | 샘플레이트 단계 (0=1kHz) |
| `transport` | 0 | 0=UDP |

> ⚠️ 라즈베리파이 IP가 바뀌면 다시 주입해야 합니다.
> 라우터에서 **DHCP 고정 할당**으로 IP를 고정해두는 것을 권장합니다.

## 4. 검증 (1kHz 수신 확인)

디바이스에 전원(USB)을 넣으면 자동으로 WiFi 연결 → 스트리밍을 시작합니다.
수신기 화면에 1초마다 다음이 출력되면 정상:

```
[192.168.0.225] 패킷=5 샘플=1000 실효레이트≈1000Hz (목표 1000Hz) 유실≈0.00%
```

**검증 통과 기준 (rate_step=0, 1kHz):**
- ✅ 실효레이트 ≈ **1000 Hz** (목표와 일치)
- ✅ 유실 ≈ **0%** (UDP라 약간의 유실은 정상, 1% 미만 권장)
- ✅ `vibration.csv` 에 `x,y,z` 값이 정상 범위로 기록됨

종료(Ctrl+C) 시 총 패킷/샘플/평균 레이트 요약이 출력됩니다.

## 동작 요약
- UDP 포트(기본 9000)로 수신 대기
- 패킷 헤더 검증(magic "IIS3") 후 샘플 파싱
- CSV로 저장: `seq, timestamp_ms, sample_idx, x, y, z`
- 1초마다 실효 샘플레이트 / 유실률 표시 (seq 기반)

## 샘플레이트 단계 (rate_step)

| rate_step | 샘플레이트 | 비고 |
|-----------|-----------|------|
| 0 | 1 kHz | 기본, **1단계 검증 대상** |
| 1 | 3.3 kHz | |
| 2 | 6.6 kHz | 고속 (FIFO 읽기 필요, 2단계) |
| 3 | 13.3 kHz | 고속 (2단계) |
| 4 | 26.6 kHz | 최대 (2단계) |

> 현재 펌웨어 1단계는 **폴링 방식**이라 1kHz까지 안정적입니다.
> 3.3kHz 이상은 FIFO 묶음 읽기(2단계) 구현 후 검증 예정.

## 패킷 포맷
펌웨어 `components/sensor_streamer/sensor_streamer.h` 와 일치:
```
헤더 16B: magic(4,"IIS3") ver(1) rate_step(1) sample_count(2) seq(4) timestamp_ms(4)
페이로드: sample_count × {int16 x, int16 y, int16 z}  (little-endian)
```
- mg 변환은 수신 측에서 (감도 = 풀스케일 의존)
- seq로 패킷 손실/유실률 측정

## 문제 해결

| 증상 | 확인 |
|------|------|
| 아무것도 안 들어옴 | 디바이스/라즈베리파이가 **같은 네트워크(2.4GHz)** 인지, NVS의 `srv_ip`가 위 표시된 IP와 같은지 |
| 방화벽 | `sudo ufw allow 9000/udp` (ufw 사용 시) |
| 디바이스 WiFi 미연결 | 디바이스 시리얼 로그에서 WiFi 연결 성공 여부 확인 |
| 레이트가 1000Hz 미만 | 네트워크 혼잡 또는 WiFi 신호 약함 — 디바이스를 라우터 근처로 |
