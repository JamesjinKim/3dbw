# Plan: GUI 서버 설정 + 송수신 확인 (전체 패키지 완성)

- **Feature**: gui-server-config
- **작성일**: 2026-06-01
- **상태**: Plan
- **관련**: [[wifi-sensor-streaming]] (펌웨어 스트리밍, 검증 완료)

## 1. 배경 / 문제

실기기 최종 테스트에서 확인된 문제:
- GUI 설정 툴(config-tool)이 **WiFi(`wifi_ssid`/`wifi_pass`)만 NVS에 저장**한다.
- 펌웨어 스트리밍 시작 조건은 `WiFi 연결됨 + 서버 설정 존재(srv_ip)` 인데, GUI가 서버 설정을 저장하지 않아 **"WiFi 설정 완료"는 떠도 스트리밍이 시작되지 않음**.
- 현재는 서버 설정을 터미널에서 수동 NVS 주입으로 우회 중 → 실제 제품 플로우 미완성.

## 2. 목표 (테스터 피드백 = 전체 패키지)

설정 툴 하나로 **출고 후 사용자 첫 설정 전체 흐름**을 완료한다:

```
① WiFi 설정  →  ② 라즈베리파이(서버) IP 설정  →  ③ 데이터 송수신 확인  →  ④ 설정 화면 종료
```

이 4단계가 끝나면 사용자가 더 손댈 것 없이 제품이 동작해야 한다.

## 3. 범위 (In Scope)

| # | 요구사항 | 영역 |
|---|----------|------|
| FR-1 | NVS 생성기가 WiFi + 서버설정(srv_ip/srv_port/stream_rate/transport)을 함께 기록 | nvs.rs |
| FR-2 | `write_config` 커맨드: WiFi+서버 한 번에 NVS 주입 | lib.rs |
| FR-3 | UI에 서버 IP / 포트 / 샘플레이트 입력칸 추가 (②단계) | UI |
| FR-4 | "데이터 송수신 확인" 단계: 디바이스가 실제로 전송 시작했는지 확인 (③단계) | lib.rs + UI |
| FR-5 | 확인 완료 후 완료 화면 → "설정 종료" (④단계) | UI |

## 4. 범위 외 (Out of Scope)

- 라즈베리파이 수신기 변경 (이미 완성·검증됨)
- 펌웨어 변경 (스트리밍 동작 검증 완료, 변경 불필요)
- 고속 FIFO(2단계 스트리밍)
- TCP transport (UDP만)

## 5. 핵심 기술 과제

- **NVS 바이너리 포맷**: 현재 `add_string`(0x21), `add_namespace`(u8 0x01)만 구현됨.
  - 추가 필요: **u8 엔트리**(stream_rate, transport), **u16 엔트리**(srv_port), srv_ip는 기존 add_string 재사용.
  - ESP-IDF `nvs_partition_gen.py` 출력과 **byte-exact 일치** 유지 필수 (기존 단위테스트 패턴 확장).
- **송수신 확인(FR-4)**: 디바이스→라즈베리파이는 직접 확인 불가(서로 다른 망 경로). 대신 **디바이스 시리얼 로그에서 "스트리밍 시작" 또는 전송 통계(패킷>0)** 를 읽어 확인하는 방식이 현실적.
  - 대안: GUI가 잠깐 UDP 포트를 열어 자기 자신으로 오는지 보는 건 불가(목적지가 라즈베리파이). → 시리얼 로그 확인이 정답.

## 6. 성공 기준

- [ ] GUI에서 WiFi+서버 입력 → 저장 → 디바이스 NVS에 6개 키 모두 기록됨
- [ ] 생성된 NVS가 ESP-IDF 기준 bin과 byte-exact 일치 (단위테스트 통과)
- [ ] 디바이스 재부팅 시 스트리밍 자동 시작 (시리얼 "스트리밍 시작" 로그)
- [ ] GUI가 "송수신 확인됨"을 표시하고 종료 단계로 진행
- [ ] 라즈베리파이에 데이터 도착 (수동 주입 없이 GUI만으로)

## 7. 리스크

| 리스크 | 대응 |
|--------|------|
| NVS u8/u16 엔트리 포맷 오류 → 디바이스가 설정 못 읽음 | nvs_partition_gen.py 기준 bin과 byte-exact 비교 테스트 |
| 송수신 확인을 시리얼로 하는데 포트 경합 | 확인 시에만 짧게 시리얼 open, 즉시 close |
| 기존 WiFi 전용 테스트(reference bin) 깨짐 | 기존 테스트 유지 + 신규 full-config 테스트 별도 추가 |
