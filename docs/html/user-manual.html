IIS3DWB 무선 진동 모니터링 시스템

사용자 설명서 — 센서 디바이스(ESP32-S3) + 라즈베리파이 수신 서버
실기기 검증 완료 펌웨어 1단계 (1 kHz) UDP 무선 스트리밍 ESP-IDF v5.4.3 ESP32-S3

1. 시스템 개요

IIS3DWB 초광대역 진동 센서가 측정한 가속도 데이터를 USB로 전력만 공급받은 채 WiFi로 무선 전송하고, 같은 네트워크의 라즈베리파이가 이를 수신·기록하는 시스템입니다.

IIS3DWB진동 센서
→ SPI →
ESP32-S3WiFi 송신 (UDP)
→ WiFi →
라즈베리파이수신 · CSV 저장

① 센서 디바이스 영역 펌웨어

  • SPI로 센서 가속도값 읽기
  • NVS에 저장된 WiFi로 자동 연결
  • NVS의 서버 IP로 UDP 패킷 전송
  • 부팅 시 전 과정 자동 시작

② 수신 서버 영역 라즈베리파이

  • UDP 포트로 패킷 수신
  • 헤더 검증 + 샘플 파싱
  • 실시간 X/Y/Z 값 화면 표시
  • 전체 샘플 CSV 로그 저장
핵심 동작 원리: USB는 전력 공급 전용입니다. 측정 데이터는 전부 WiFi로 무선 전송됩니다. 디바이스에 전원만 넣으면 사용자 조작 없이 자동으로 WiFi 연결 → 스트리밍이 시작됩니다.

2. 센서 디바이스 펌웨어

ESP32-S3 기반 디바이스에 탑재되는 펌웨어로, 센서 읽기·WiFi 연결·무선 전송을 담당합니다.

2.1 하드웨어 연결 (SPI 핀맵)

IIS3DWB 센서는 SPI(Mode 3, CPOL=1/CPHA=1)로 ESP32-S3와 연결됩니다.

신호ESP32-S3 GPIO설명
MOSIGPIO 13Master Out Slave In
MISOGPIO 12Master In Slave Out
SCLKGPIO 14SPI 클럭 (기본 1 MHz, 최대 10 MHz)
CSGPIO 15Chip Select (하드웨어 자동 제어)
VDD / VDDIO3.0 ~ 3.6 V전원
※ 핀 번호는 idf.py menuconfig → IIS3DWB Vibration Sensor Configuration 에서 변경 가능합니다.

2.2 동작 흐름

전원 인가 시 펌웨어는 다음 순서로 자동 진행합니다:

0 NVS 설정 로드
Flash의 devcfg 영역에서 WiFi 자격증명과 서버 설정을 읽습니다.
1 WiFi 연결
저장된 SSID로 STA 모드 연결. 실패 시 점진적 간격(7~15초)으로 최대 5회 재시도.
2 센서 초기화
SPI 통신 확인 (WHO_AM_I = 0x7B), 풀스케일 ±4g, 가속도계 활성화.
3 센서 읽기 태스크
주기적으로 3축 가속도 raw 데이터 취득.
4 스트리밍 시작 (조건 충족 시)
센서 정상 + WiFi 연결됨 + 서버 설정 존재 → 무선 전송 시작.

스트리밍 내부 구조 — 생산자/소비자 2-태스크

sensor_task센서 읽기
→ 링버퍼 적재
→
링버퍼 16KB완충
→
tx_task200샘플 모아
UDP 전송
WiFi가 잠깐 느려져도 링버퍼가 완충 역할을 합니다. 가득 차면 가장 오래된 데이터를 드롭(통계 기록).
구성 컴포넌트: components/sensor_streamer (스트리밍), components/config_manager (NVS 설정), components/wifi_manager (WiFi), components/iis3dwb (센서 드라이버).

2.3 NVS 설정 항목

디바이스 동작에 필요한 설정은 Flash의 NVS(devcfg 네임스페이스)에 저장됩니다. 설정 툴 또는 NVS 직접 주입으로 기록합니다.

키타입예시설명
wifi_ssidstringMyWiFi-2.4G접속할 WiFi 이름 (2.4GHz 전용)
wifi_passstring********WiFi 비밀번호
srv_ipstring192.168.0.37라즈베리파이 IP (수신 서버)
srv_portu169000수신 포트
stream_rateu80샘플레이트 단계 (0=1kHz, 표 참고)
transportu800=UDP (1=TCP, 후속 단계)
주의: WiFi와 수신 서버(라즈베리파이)는 반드시 같은 2.4GHz 네트워크에 있어야 합니다. ESP32-S3는 5GHz를 지원하지 않습니다.

3. 라즈베리파이 수신 서버 라즈베리파이

원격지 라즈베리파이에서 디바이스가 보내는 진동 데이터를 수신·기록하는 프로그램입니다. (Python 3.6+ 표준 라이브러리만 사용, 추가 설치 불필요)

3.1 설치 (git clone)

git clone https://github.com/JamesjinKim/3dbw.git
cd 3dbw/rpi-receiver
chmod +x run.sh        # 최초 1회

3.2 실행 및 화면 보기

./run.sh                 # 포트 9000, vibration.csv 에 저장
./run.sh --fs 4          # 센서 풀스케일 ±4g (mg 환산용, 펌웨어와 일치)
PORT=9001 ./run.sh       # 포트 변경

실행하면 먼저 이 라즈베리파이의 IP를 표시합니다. 이 값을 디바이스 NVS의 srv_ip에 넣어야 합니다:

====================================================
  IIS3DWB UDP 수신기
====================================================
  이 라즈베리파이 IP : 192.168.0.37
  수신 포트          : 9000

  ▶ 디바이스(ESP32) NVS 에 아래 값을 넣으세요:
      server_ip   = 192.168.0.37
      server_port = 9000
====================================================
  저장 파일 : vibration.csv  (raw + mg 전체 샘플)
  풀스케일  : ±4g  (감도 0.122 mg/LSB)
[수신 대기] … 센서를 흔들면 X/Y/Z 값이 변합니다. (Ctrl+C 종료)

데이터가 들어오면 1초마다 실시간 가속도값이 갱신됩니다:

[192.168.0.225] 1000Hz 유실0.00% │ X=  +12.3 Y=   -8.7 Z= +1001.4 mg │ 샘플=10000
표시 항목의미
[192.168.0.225]송신한 디바이스 IP
1000Hz순간 수신 레이트 (직전 1초 구간)
유실0.00%패킷 손실률 (seq 기반)
X / Y / Z … mg실제 센서 가속도값 (최신 샘플, mg 단위)
샘플=10000누적 수신 샘플 수
센서가 살아있다는 증거: 센서를 가만히 두면 한 축(보통 Z)이 중력 ≈ 1000 mg (1g) 근처로 나옵니다. 센서를 흔들거나 두드리면 X/Y/Z 값이 출렁입니다.

3.3 CSV 로그 형식 (vibration.csv)

수신한 모든 샘플이 CSV로 저장됩니다 (raw LSB + mg 가속도 동시 기록).

컬럼설명
recv_iso라즈베리파이 수신 시각 (예: 2026-06-01T08:52:32.800)
seq패킷 시퀀스 번호 (유실 추적용)
timestamp_ms디바이스 부팅 후 경과 ms (패킷 생성 시각)
sample_idx패킷 내 샘플 순번 (0~199)
x_raw, y_raw, z_raw원시 가속도 (int16 LSB)
x_mg, y_mg, z_mgmg 단위 가속도 (감도 환산)
후처리(FFT, RMS 등)는 x_mg/y_mg/z_mg를, 정밀 분석은 x_raw 등을 사용하세요.

4. 통신 프로토콜 (패킷 포맷)

디바이스와 수신기는 다음 바이너리 UDP 패킷으로 통신합니다. (little-endian)

구간크기필드값/설명
헤더4 Bmagic0x49495333 ("IIS3")
1 Bversion프로토콜 버전 = 1
1 Brate_step샘플레이트 단계 0~4
2 Bsample_count이 패킷의 샘플 수 (200)
4 Bseq패킷 시퀀스 번호
4 Btimestamp_ms디바이스 부팅 후 ms
페이로드6 B × Nx, y, zint16 가속도 × 샘플 수
패킷 크기 = 16 B 헤더 + 200 × 6 B = 1216 B. 헤더 정의는 펌웨어 sensor_streamer.h와 수신기 udp_receiver.py가 바이트 단위로 일치합니다.

5. 샘플레이트 단계

NVS의 stream_rate 값으로 전송 속도를 5단계로 설정합니다.

stream_rate샘플레이트상태
01 kHz✅ 검증 완료 (기본, 1단계)
13.3 kHz2단계 (FIFO) 예정
26.6 kHz2단계 예정
313.3 kHz2단계 예정
426.6 kHz2단계 (최대) 예정
현재 1단계는 폴링 방식이라 1 kHz까지 안정적입니다. 3.3 kHz 이상 고속 전송은 센서 내장 FIFO 묶음 읽기(2단계) 구현 후 지원됩니다.

6. 전체 검증 절차

1 라즈베리파이 수신기 실행 RPi
./run.sh → 표시되는 IP를 메모.
2 디바이스에 설정 주입 디바이스
NVS에 srv_ip(위 IP), srv_port=9000, stream_rate=0, WiFi 정보 기록 후 펌웨어 플래시.
3 디바이스에 전원 인가 (USB) 디바이스
자동으로 WiFi 연결 → 스트리밍 시작. (시리얼 로그로 확인 가능)
4 수신 화면 확인 RPi
X/Y/Z 값이 갱신되고, 정지 시 한 축이 ±1000 mg(중력) 근처면 정상.
검증 통과 기준 (1 kHz):
  • 순간 레이트 ≈ 1000 Hz
  • 유실 ≈ 0% (1% 미만 권장)
  • 정지 시 한 축이 ±1000 mg(중력) 근처, 움직이면 값 변동
  • vibration.csv에 전체 샘플 기록
실기기 검증 결과: 디바이스 송신 1000 Hz / 드롭 0, 라즈베리파이 수신 유실 0% — 정상 동작 확인 완료.

7. 문제 해결

증상확인 사항
수신기에 아무것도 안 옴디바이스·라즈베리파이가 같은 2.4GHz 네트워크인지, NVS의 srv_ip가 수신기가 표시한 IP와 같은지
방화벽이 막음sudo ufw allow 9000/udp (ufw 사용 시)
디바이스 WiFi 미연결시리얼 로그에서 "WiFi Connected" 확인. SSID/비밀번호, 5GHz 여부 확인
레이트가 1000 Hz 미만WiFi 신호 약함/혼잡 — 디바이스를 라우터 근처로. (단, 누적평균은 대기시간 영향 → 순간 레이트로 판단)
센서 값이 0 또는 이상SPI 배선(MOSI/MISO/SCLK/CS), 전원 전압, SPI Mode 3 확인
라즈베리파이 IP가 바뀜라우터에서 DHCP 고정 할당 권장. 바뀌면 디바이스 NVS 재주입 필요
