🛠️ IIS3DWB 센서 설정 툴 (Python)

PC에서 센서의 WiFi·수신서버·측정속도·읽기방식을 입력해 디바이스에 저장하는 설정 도구
무설치 (tkinter 표준) ESP-IDF 전체 설치 불필요 esptool 만 필요 Mac · Linux(RPi) · Windows

목차

1. 이 툴은 무엇인가

IIS3DWB 진동 센서 디바이스(ESP32-S3)는 센서 펌웨어가 이미 플래시되어 있습니다. 이 툴은 그 펌웨어가 부팅 시 읽는 설정값(WiFi, 수신 서버 IP, 측정 속도 등)을 디바이스의 Flash 메모리(NVS = EEPROM 역할)에 써넣는 역할만 합니다.

핵심: 설정값을 한 번 써두면 디바이스는 전원을 넣을 때마다 그 값을 읽어 자동으로 WiFi에 연결하고 데이터를 전송합니다. 값이 바뀔 때(WiFi 변경, 수신 서버 IP 변경 등) 이 툴로 다시 써넣기만 하면 됩니다.
이 툴이 하지 않는 것: 센서 펌웨어 자체를 빌드하거나 플래시하지 않습니다. 펌웨어는 별도로 플래시되어 있어야 합니다 (펌웨어 저장소: github.com/JamesjinKim/3dbw → idf.py flash).

3. 준비물

항목설명
Python 3.7+표준 라이브러리 tkinter 사용 (대부분 기본 포함)
esptool둘 중 하나면 됨: ① ESP-IDF 활성화(source .../export.sh) 또는 ② pip install esptool
(선택) pyserialUSB 포트 자동 인식에 사용. 없으면 /dev 스캔으로 폴백. pip install pyserial
디바이스펌웨어가 플래시된 IIS3DWB 디바이스를 데이터용 USB 케이블로 연결
USB 케이블 주의: 충전 전용 케이블은 데이터 통신이 안 됩니다. 주입 시 No serial data received 오류가 나면 케이블부터 의심하세요.

4. 실행 방법

이 폴더(set_sensor_gui.py + nvs_gen.py)만 PC로 복사하면 됩니다.

# esptool 이 없으면 둘 중 하나
source ~/esp/v5.4.3/esp-idf/export.sh      # ESP-IDF 활성화 (경로는 환경에 맞게)
#   또는:  pip install esptool

# 설정 툴 실행
cd sensor-setup-py
python3 set_sensor_gui.py
라즈베리파이(헤드리스/SSH)에서: GUI 창이 떠야 하므로 모니터가 연결돼 있거나, SSH 접속 시 ssh -X (X11 포워딩) 로 접속해야 화면이 보입니다.

데이터 읽기 방식 (read_mode)

선택값설명
인터럽트 · 저부하1기본 — INT 신호로 필요할 때만 깨어나 CPU 부하 최소. 신호 미수신 시 자동 폴링 fallback (실기기 26.6kHz 검증).

두 방식 모두 26.6kHz 무손실로 실기기 검증됨. 잘 모르겠으면 기본값 그대로 두면 됩니다.

7. 여러 센서 연속 설정

입력값은 자동으로 기억됩니다 (이 PC의 ~/.iis3dwb_sensor_setup.json). 같은 WiFi·같은 라즈베리파이로 여러 대를 빠르게 등록할 수 있습니다.

흐름: 1번 센서 입력·주입 → 입력값 기억됨 → 2번 센서는 USB 케이블만 바꿔 꽂고 "포트 새로고침" 후 바로 "주입" → 3번 센서도 동일… (앱을 껐다 켜도 입력값 유지)

8. 동작 확인

주입 후 디바이스 시리얼 로그로 연결·전송을 확인할 수 있습니다.

idf.py -p /dev/ttyACM0 monitor      # 포트는 환경에 맞게 (종료: Ctrl+])

로그에서 다음을 확인:

  • WiFi Connected + IP : 192.168.0.xxx
  • 📡 센서 데이터 스트리밍 중! → (라즈베리파이IP):9000
  • [스트리밍] 패킷=... 드롭=0 에러=0

라즈베리파이 수신기 화면에 실시간 가속도값(X/Y/Z mg)과 실효 레이트가 표시되면 성공입니다.

9. 문제 해결

가장 흔한 원인 = 수신 서버 IP 불일치. "주입은 됐는데 라즈베리파이에 아무것도 안 온다"면 대개 디바이스에 넣은 IP ≠ 라즈베리파이 현재 IP 입니다. 라즈베리파이에서 hostname -I 로 현재 IP를 확인하고 그 IP로 다시 주입하세요. (라즈베리파이 IP를 고정하면 근본 해결됩니다.)
증상해결
esptool 못 찾음 / 관련 오류pip install esptool 또는 ESP-IDF 활성화. (NVS 생성은 ESP-IDF 불필요, 굽기만 필요)
No serial data received① 데이터용 USB 케이블인지 확인(충전 전용 ✗) ② 다른 프로그램(idf.py monitor 등)이 포트 점유 중인지 확인 후 닫기 ③ 케이블 다시 꽂고 "포트 새로고침"
포트 목록이 비어 있음USB 연결 확인 후 "포트 새로고침". Linux 는 dialout 그룹 권한 필요 (sudo usermod -a -G dialout $USER 후 재로그인). 인식이 약하면 pip install pyserial.
GUI 창이 안 뜸 (라즈베리파이)모니터 연결 또는 ssh -X 로 접속. 헤드리스 SSH 는 화면 출력이 없음.
주입 후 수신 안 됨위 경고 박스 참고 — RPi IP 불일치 확인 후 재주입. 디바이스·RPi 가 같은 2.4GHz 네트워크인지도 확인.

10. NVS 규격 (참고 · 수정 금지)

항목값
NVS namespacedevcfg
NVS 파티션 offset / size0x9000 / 0x6000 (24576 B)
키 순서wifi_ssid, wifi_pass, srv_ip, srv_port(u16), stream_rate(u8), transport(u8), read_mode(u8)
이 규격은 펌웨어 및 Tauri 설정툴(config-tool)과 동일합니다. nvs_gen.py 는 그 Rust 구현을 파이썬으로 1:1 포팅한 것으로, 검증된 기준 바이너리와 byte-exact 일치가 확인되어 있습니다.
보안 주의: 입력값 기억 파일(~/.iis3dwb_sensor_setup.json)에 WiFi 비밀번호가 평문으로 저장됩니다(이 PC 안에만). 공용 PC 라면 작업 후 rm ~/.iis3dwb_sensor_setup.json 으로 삭제하세요.
